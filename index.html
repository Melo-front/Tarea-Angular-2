Login en Angular

Hacer el Login en Angular

Un login es el proceso mediante el cual un usuario ingresa sus credenciales (usuario y contraseña) para acceder a una aplicación. En Angular, generalmente se crea un formulario que captura estos datos, los envía a un servidor mediante un servicio HTTP y recibe una respuesta indicando si la autenticación fue exitosa.

Pasos básicos para implementar un login:

  1. Crear un componente de login.
  2. Crear un formulario con los campos de usuario y contraseña.
  3. Validar los datos ingresados.
  4. Enviar las credenciales al backend mediante un servicio.
  5. Recibir un token de autenticación (JWT, por ejemplo).
  6. Guardar el token en el navegador.
  7. Redirigir al usuario a una vista autorizada.

Ejemplo

Ejemplo de formulario


<form (ngSubmit)="login()">
  <input type="text" [(ngModel)]="usuario" name="usuario" placeholder="Usuario">
  
  <input type="password" [(ngModel)]="password" name="password" placeholder="Contraseña">
  
  <button type="submit">Ingresar</button>
</form>

Vistas Protegidas

¿Qué son las Vistas Protegidas?

Las vistas protegidas son componentes o páginas cuyo acceso está restringido únicamente a usuarios autenticados o con permisos específicos.
En aplicaciones reales existen páginas que no deben estar disponibles para todos los usuarios, como paneles administrativos, reportes o configuraciones.

Ejemplos de vistas protegidas

Vista Usuario normal Administrador
Inicio Sí Sí
Perfil Sí Sí
Inventario No Sí
Gestión de usuarios No Sí
Reportes financieros No Sí

¿Por qué son importantes?

Seguridad
Evitan accesos no autorizados.
Protección de datos
Restringen información sensible.
Control de permisos
Permiten diferenciar roles.
Implementación
Las vistas protegidas se configuran en el archivo de rutas:

Ejemplo


{
 path: 'dashboard',
 component: DashboardComponent,
 canActivate: [authGuard]
}

Guards en Angular

¿Qué son los Guards en Angular?

Los Guards son mecanismos que Angular ejecuta antes de permitir la navegación entre rutas. Su función es controlar quién puede acceder a determinadas páginas.

¿Por qué son necesarios?

Ocultar un menú no impide que un usuario escriba directamente una URL en el navegador. Los Guards validan los permisos antes de cargar la ruta.

Ejemplos de vistas protegidas

Guard Función
CanActivate Controla el acceso a una ruta.
CanActivateChild Protege rutas hijas.
CanDeactivate Evita salir de una página.
Resolve Obtiene datos antes de cargar una vista.
CanMatch Determina si una ruta puede ser utilizada.

Beneficios

  • Mayor seguridad.
  • Control de roles.
  • Protección de rutas sensibles.
  • Mejor organización del código.

CanActivate

Es el más utilizado para proteger rutas


import { CanActivateFn } from '@angular/router';

export const authGuard: CanActivateFn = () => {
  return localStorage.getItem('token') != null;
};

Uso del Guard

Si existe un token, Angular permitirá el acceso; de lo contrario, bloqueará la navegación.


{
  path: 'dashboard',
  component: DashboardComponent,
  canActivate: [authGuard]
}

Uso del Guard


ng generate guard auth

o 

ng g guard auth

Interpolar

¿Qué es Interpolar?

La interpolación es un mecanismo utilizado por Angular para mostrar datos dinámicos del componente dentro de una plantilla HTML.
Utiliza la sintaxis: 


{{ expresion }}

Ejemplo básico

TypeScript:


nombre = "Juan";

HTML:


<h1>Bienvenido {{ nombre }}</h1>

Resultado:


Bienvenido Juan

Interpolación con expresiones:


<p>{{ 10 + 5 }}</p>

Resultado:


15

Interpolación con métodos


obtenerNombre(){
  return "Carlos";
}

HTML

{{ obtenerNombre() }}

Casos de uso

  • Mostrar nombres de usuarios.
  • Mostrar precios.
  • Mostrar fechas.
  • Mostrar estados de procesos.
  • Mostrar resultados calculados.

¿Qué es ngIf?

*ngIf es una directiva estructural que permite mostrar o eliminar elementos del DOM dependiendo de una condición.

Sintaxis


<div *ngIf="condicion">
  Contenido visible
</div>

Ejemplo

TypeScript


logueado = true;

HTML


<p *ngIf="logueado">
  Usuario autenticado
</p>

Resultado


Usuario autenticado

Ejemplo con else


<div *ngIf="logueado; else acceso">
  Bienvenido al sistema
</div>

<ng-template #acceso>
  Debe iniciar sesión
</ng-template>

Ventajas

  • Mejora el rendimiento.
  • Oculta contenido innecesario.
  • Controla la visualización de elementos.
  • Elimina elementos del DOM cuando no son necesarios.

¿Qué es ngFor?

*ngFor es una directiva estructural utilizada para recorrer arreglos o colecciones y generar contenido HTML dinámicamente.

Sintaxis


<li *ngFor="let item of lista">
  {{ item }}
</li>

Ejemplo

TypeScript


usuarios = [
  'Juan',
  'Ana',
  'Pedro'
];

HTML


<ul>
  <li *ngFor="let usuario of usuarios">
    {{ usuario }}
  </li>
</ul>

Resultado


Juan
Ana
Pedro

Obtener índice


<li *ngFor="let usuario of usuarios; let i = index">
  {{ i }} - {{ usuario }}
</li>
